# EE115B Lecture2 Part2 — Codes & Error Detection 数制编码与检错

<aside>
🔢

**本节主题：** 有符号数 2's complement（二进制补码）与 overflow（溢出）判定 → BCD（Binary Coded Decimal 二-十进制编码）与其加法校正 → Gray code（格雷码）→ ASCII 字符编码 → error detection（检错）三法：parity（奇偶校验）/ CRC / Hamming code（汉明码）。

**教材：** Floyd《Digital Fundamentals》Ch.2（含 TABLE 2–7 ASCII 表）。讲义来源 EE115B（Chenxi Xiao），本节为 **Lecture 2 Part 2**，共 23 页。

**核心脉络：** ① 有符号数：补码表示 $N^*=2^n-N$ + 减法即「加补码」+ 溢出判定 → ② BCD：8421 编码 + 「逢非法加 6」加法 + Excess-3 等其它码 → ③ Gray code：unweighted（无权）、相邻只变 1 位、编码盘应用 → ④ ASCII：7-bit 字符编码 + 大小写位运算 → ⑤ 检错：parity → CRC → Hamming code（可纠错）。

</aside>

## 1️⃣ 数制转换总览（Conversions Recap）

<aside>
🧭

本页是上一节的收口：要能在 2 / 8 / 10 / 16 进制之间自由互转。**核心思想：8 进制 ↔ 3 位二进制、16 进制 ↔ 4 位二进制；十进制用「按权展开」进、用「除基取余」出。**

</aside>

| 转换方向 | 方法 |
| --- | --- |
| 2 → 8 / 2 → 16 | 从右往左每 **3 位 / 4 位**一组，每组直接写成 1 位八 / 十六进制 |
| 8 → 2 / 16 → 2 | 每位八 / 十六进制 → **3 位 / 4 位**二进制 |
| 8 ↔ 16 | 没有直通路，**经二进制中转** |
| 2/8/16 → 10 | 按权展开求和：$\sum_i d_i \cdot r^i$（$r$ 为基数） |
| 10 → 2/8/16 | 整数部分**除基取余**（余数倒序）；小数部分**乘基取整**（整数正序） |

## 2️⃣ 二进制补码 2's Complement

<aside>
💡

**定义：** 一个负数用「其正数的 bitwise NOT（按位取反，即 ones' complement 反码）再加 1」表示。等价公式：$N^* = 2^n - N$（$n$ 为位宽）。

**8-bit 范围：** 可表示 256 个数，$-128 \sim +127$。最高位是符号位（0 正 1 负）。

</aside>

**取补码的两步法：** ① 每一位取反（$0\leftrightarrow1$）得到反码；② 末位 $+1$。例：$+3 = 00000011 \Rightarrow$ 反码 $11111100 \Rightarrow +1 \Rightarrow 11111101 = -3$。

**8-bit 补码对照（代表行）：**

| Binary | 补码解释 | 无符号解释 |
| --- | --- | --- |
| 00000000 | 0 | 0 |
| 01111111 | +127 | 127 |
| 10000000 | −128 | 128 |
| 11111110 | −2 | 254 |
| 11111111 | −1 | 255 |
- 📚 拓展 — 为什么硬件偏爱 2's complement？
    
    ① **加减统一**：$A-B = A + (-B)$，减法直接复用加法器，不必单独做减法电路。
    ② **零唯一**：不像 sign-magnitude（原码）/ ones' complement（反码）有 $+0$ 和 $-0$ 两个零，补码只有一个 0，省一个状态、比较更简单。
    ③ **溢出易判**：只看符号位是否「该对却错」即可（见下一节）。这就是几乎所有 CPU 用补码存整数的原因。
    

### 减法示例（Subtraction）

**思路：** 减一个数 = 加它的 2's complement。例：$(+8)-(+3) = (+8)+(-3)$：

```
  00001000      Minuend (+8)
+ 11111101      2's complement of subtrahend (−3)
-----------
1 00000101      丢弃进位(discard carry) → Difference = +5
```

⚠️

**关键：必须事先确定位宽（number of bits）。** 位数不够会把正确结果误判成溢出（见下表 $+13+10=+23$ 在 4-bit 下放不下）。下图右两列就是「位宽不足 vs 充足」的对照。

</aside>

### 溢出 Overflow

<aside>
🎯

**溢出判定：** 当两个数相加、结果所需位数超过给定位宽时发生溢出，表现为**符号位错误**。
**只可能在「两数同号」时发生**（同正得负 / 同负得正）；异号相加绝不溢出。判据：**结果符号位与两加数符号位不同 → 溢出**。

</aside>

**例：** $125 + 58 = 183$，但 8-bit 有符号上限是 $+127$：$01111101 + 00111010 = 10110111$，符号位变 1（错误），数值也错。

*📄 见文末「原始 Slides」第 5 页：Overflow 溢出判定示例（*$125+58$*，标注 Sign / Magnitude incorrect 的对比）。*

- 📚 拓展 — 怎样「自信地」算而不踩溢出？
    
    ① **预留足够位宽**：结果位宽 ≥ 两加数位宽 + 1。
    ② **符号扩展（sign extension）**：把短补码扩到更宽时，用符号位填充高位（正数补 0、负数补 1），数值不变。
    ③ **检查 carry-in 与 carry-out**：补码加法中，若进入符号位的进位 ≠ 移出符号位的进位，则溢出（硬件常用此法置 overflow flag）。
    

## 3️⃣ BCD 二-十进制编码（Binary Coded Decimal）

<aside>
💡

**BCD：** 把十进制每一位数字 $0\sim9$ 各用 **4 位二进制**单独编码。最常用的是 **8421 BCD**——4 位权重为 $2^3,2^2,2^1,2^0 = 8,4,2,1$。

**非法码：** $1010,1011,1100,1101,1110,1111$（即 $10\sim15$）在 8421 BCD 中**无效**，因为十进制每位只到 9。

</aside>

**为什么要 BCD？** 直接用纯二进制对显示系统（如 7-segment display 数码管）不友好；用 BCD + Display Decoder（显示译码器，如 CD4543B）可把每个十进制位直接驱动一位数码管。

*📄 见文末「原始 Slides」第 6 页：BCD 显示译码框图（7-segment display + Display Decoder CD4543B + BCD-to-7seg 译码框图）。*

**列权（column weights）求值：** 把每段 4 位按 $\dots 8000\,4000\,2000\,1000\;800\,400\,200\,100\;80\,40\,20\,10\;8\,4\,2\,1$ 加权。例 $1000\,0011\,0101\,1001$ → $8000+200+100+40+10+8+1 = 8359$。

**十进制 → BCD（按位拆）：** 每个十进制位换成 4 位即可，例：

```
35   → 0011 0101
98   → 1001 1000
170  → 0001 0111 0000
2469 → 0010 0100 0110 1001
```

### 🔧 BCD 加法（含「加 6」校正）

<aside>
🧠

**三步法：** ① 先按普通二进制把两个 BCD 相加；② 若某 4 位组结果 ≤ 9 → 合法，保留；③ 若该组 **> 9 或产生进位** → 非法，**加 6（0110）**跳过 6 个非法态回到 8421；加 6 若再生进位，就并入下一组。

</aside>

*📄 见文末「原始 Slides」第 11 页：8421 BCD 加法 worked examples（a–d 四组，含「逢非法加 6（0110）」校正过程）。*

### 其它 BCD 码（Alternative BCD Codes）

**Excess-3（余 3 码）：** 在原 BCD 值上 $+3$，使编码 **reflexive（自反 / 镜像）**；它和 5211 码都是 **self-complementing（自补）**码——对 9 求补只需按位取反，便于减法。

| Decimal | 7421 | 5421 | 5211 | 2421 | Excess-3 |
| --- | --- | --- | --- | --- | --- |
| 0 | 0000 | 0000 | 0000 | 0000 | 0011 |
| 1 | 0001 | 0001 | 0010 | 0001 | 0100 |
| 2 | 0010 | 0010 | 0011 | 0010 | 0101 |
| 3 | 0011 | 0011 | 0101 | 0011 | 0110 |
| 4 | 0100 | 0100 | 0111 | 0100 | 0111 |
| 5 | 0101 | 1000 | 1000 | 1011 | 1000 |
| 6 | 0110 | 1001 | 1010 | 1100 | 1001 |
| 7 | 1000 | 1010 | 1100 | 1101 | 1010 |
| 8 | 1001 | 1011 | 1101 | 1110 | 1011 |
| 9 | 1010 | 1100 | 1111 | 1111 | 1100 |
- 📚 拓展 — self-complementing（自补码）为什么好用？
    
    自补码满足：数字 $d$ 的编码按位取反，正好等于 $9-d$ 的编码。
    这样做**九的补数（9's complement）减法**时，取反电路就能直接得到补数，省去额外查表 / 运算——在早期十进制运算器里能简化减法硬件。Excess-3、2421、5211 都具备这个性质，而 8421 不具备。
    

## 4️⃣ 格雷码 Gray Code

<aside>
💡

**两大特征：** ① **unweighted（无权码）**——每位没有固定权重，不能按权直接算十进制；② **相邻码字只变化 1 位（single-bit change）**。正因如此特别适合 **shaft position encoder（轴 / 旋转位置编码器）**。

</aside>

**4-bit Gray code 表：**

| Dec | Binary | Gray | Dec | Binary | Gray |
| --- | --- | --- | --- | --- | --- |
| 0 | 0000 | 0000 | 8 | 1000 | 1100 |
| 1 | 0001 | 0001 | 9 | 1001 | 1101 |
| 2 | 0010 | 0011 | 10 | 1010 | 1111 |
| 3 | 0011 | 0010 | 11 | 1011 | 1110 |
| 4 | 0100 | 0110 | 12 | 1100 | 1010 |
| 5 | 0101 | 0111 | 13 | 1101 | 1011 |
| 6 | 0110 | 0101 | 14 | 1110 | 1001 |
| 7 | 0111 | 0100 | 15 | 1111 | 1000 |

**为什么要用 Gray code？** 普通二进制在相邻状态切换时可能**多位同时翻转**，物理上各位不可能严格同时变化，编码盘读数会瞬间出现错误中间值（glitch）。Gray code 每步只变 1 位，从根本上消除这种误读。

*📄 见文末「原始 Slides」第 15 页：旋转编码盘 Binary code vs Gray code（IR emitter/detector 读盘对比 + 机器人关节编码器图）。*

- 📚 拓展 — Binary ↔ Gray 互转公式 & Leetcode 联系
    
    **Binary → Gray：** $g_i = b_i \oplus b_{i+1}$（最高位不变，其余位 = 该位 XOR 高一位）。等价写法 $G = B \oplus (B \gg 1)$。
    **Gray → Binary：** $b_i = g_i \oplus b_{i+1}$（从最高位往低位逐位异或累加）。
    **Leetcode 联系：** 「n-bit gray code sequence」要求 $2^n$ 个数构成的序列，相邻（含首尾）只差 1 位——本质就是按 $i \oplus (i\gg1)$ 生成。这把课堂概念和算法题打通了。
    

## 5️⃣ ASCII

<aside>
💡

**ASCII：** 字母数字 + 控制字符的编码。原始用 **7-bit 编码 128 个**字符，前 32 个是 control characters（控制字符，源自电传打字机）。1981 年 IBM 推出 **8-bit extended ASCII（256 个）**；之后又有 Unicode 处理非英文字符。

</aside>

*📄 见文末「原始 Slides」第 17 页：ASCII 完整表（Floyd TABLE 2–7，含 Name/Dec/Binary/Hex 与 Graphic Symbols）。常用记忆点：`space`=32、`0`=48、`A`=65、`a`=97。*

**常考练习（exercises）：** 值↔字符互转、大小写互转、判断是否字母、统计区间字符数。Python 用 `ord()`（字符→值）和 `chr()`（值→字符）：

```python
ord('p')           # 112
chr(65)            # 'A'
chr(120)           # 'x'
chr(ord('S') + 1)  # 'T'
```

<aside>
🧠

**大小写关键位：** `'a'=97=0x61`、`'A'=65=0x41`，差 32 = $2^5$ = **bit 5**。
· 小写→大写：清除 bit 5 → `c & ~0x20`（或 $c-32$）。
· 大写→小写：置位 bit 5 → `c | 0x20`（或 $c+32$）。这正好衔接上一节 AND 清位 / OR 置位的 bit masking。
· `'a'` 到 `'m'` 共 $109-97+1 = 13$ 个字符。

</aside>

## 6️⃣ 错误检测 Error Detection

### ⑴ 奇偶校验 Parity Method

<aside>
🎯

一组位中 1 的个数要么偶、要么奇。**even parity（偶校验）**让总 1 数为偶、**odd parity（奇校验）**让总 1 数为奇——靠追加 1 个 parity bit（校验位）实现。
**能力边界：** 只能检出**单个**比特错误，**无法检出 2 个**同时出错（也不能定位 / 纠正）。校验位本质 = 所有数据位的 XOR（偶校验）。

</aside>

### ⑵ 循环冗余校验 CRC（Cyclic Redundancy Check）

- CRC 能在较大数据块里检出**多个错误**。发送端在数据块后附加一个 checksum；接收端重新计算并与收到的 checksum 比较，相同则认为无错。数据 + 校验和合称 **codeword（码字）**。

*📄 见文末「原始 Slides」第 21 页：CRC 框图（Message 经 CRC Algorithm 生成 CRC Checksum，二者拼成 Codeword）。*

- 📚 拓展 — CRC 到底怎么算？
    
    CRC 把数据看成多项式，用一个约定的 **generator polynomial（生成多项式）**做**模 2 除法**（异或代替减法）：把数据左移 $r$ 位后除以生成多项式，**余数**就是 $r$ 位 CRC 校验和，附在数据后。接收端对整个 codeword 再除一次，余数为 0 则无错。比单纯 parity 强很多，是以太网 / 存储里实际在用的检错法。
    

### ⑶ 汉明码 Hamming Code

<aside>
⭐

Hamming code 是一种 **error-correcting code（纠错码）**——不只检错，还能**定位并纠正单比特错误**。由 Richard Hamming 于 1950s 提出，在原数据里插入若干 parity bits 实现。

</aside>

**放置规则：** 校验位放在位号为 **2 的幂**的位置（$r_1,r_2,r_4,r_8\dots$ 即第 1、2、4、8 位），其余位放数据。下图演示数据 `1001101` 逐步插入 $r_1,r_2,r_4,r_8$ 得到码字 `10011100101`：

*📄 见文末「原始 Slides」第 23 页：Hamming code worked example（Data `1001101` 逐步插入* $r_1,r_2,r_4,r_8$ *→ Code `10011100101`）。*

- 📚 拓展 — 需要几个校验位？纠错怎么定位？
    
    **校验位个数** $r$ 需满足 $2^r \ge m + r + 1$（$m$ 为数据位数）——保证能覆盖「无错」+「每一位单错」共 $m+r+1$ 种情形。
    **定位错误：** 每个校验位负责一组特定位置（按位号二进制展开决定）。接收端重算各校验位，把结果按权拼成一个数 = **syndrome（校验子）**；syndrome 若为 0 表示无错，否则它的数值**正好是出错位的位号**，取反那一位即纠正。
    

## ✅ 本节总结（口诀）

<aside>
📌

1. **补码：** 取反加一、$N^*=2^n-N$；减法=加补码；8-bit 范围 $-128\sim+127$。
2. **溢出：** 只在同号相加时发生，结果符号位「该对却错」即溢出。
3. **BCD：** 每位十进制 → 4 位；加法「逢 >9 或进位就 +6」；Excess-3 自补。
4. **Gray：** 无权、相邻只变 1 位；$G=B\oplus(B\gg1)$；用于编码盘防误读。
5. **ASCII：** 7-bit 128 字符；大小写差 bit 5（0x20）。
6. **检错：** parity 检 1 位、CRC 检多位、Hamming 可纠 1 位（校验位在 2 的幂位置）。
</aside>

# next-state 表：现态/次态方向写反

状态: 🟡 订正中
知识点: FSM · 状态转移表（Sequential Logic）
错误类型: 🔄 方向/顺序写反

## ❌ 我当时怎么做的

画 next-state table 时把**现态（present state）和次态（next state）的方向弄反了** —— 把带上标加号的 $Q_1^{+}Q_0^{+}$ 当成「现在的状态」去当输入，又拿不带加号的 $Q_1 Q_0$ 去填「下一拍变成的状态」，整张表的因果方向写反。

## ✅ 正确做法

约定（present(现态) / next(次态) 的标准记法）：

- **现态 =**  $Q_1 Q_0$：这一拍时钟沿到来**之前**，寄存器里已经存着的状态 → 是表格的**输入 / 索引列**（不带 `+`）。
- **次态 =**  $Q_1^{+}Q_0^{+}$：经过 next-state logic 加时钟沿之后，寄存器在**下一拍**将变成的状态 → 是表格的**输出列**（带 `+`）。
- 因果方向永远是 **现态 → 次态**：

$$
Q_1 Q_0\ \text{(present)}\ \xrightarrow[\;\text{clock edge}\;]{\;\text{next-state logic}\;}\ Q_1^{+} Q_0^{+}\ \text{(next)}
$$

<aside>
🐈

**读表口诀：带加号的是「结果 / 下一拍」，不带加号的是「原因 / 现在」。上标**  $+$  **= next。箭头只能从无** $+$ **指向有** $+$**，绝不反。**

</aside>

## 🐈 防错提醒（下次怎么不再错）

1. **先写死表头再填表**：左列标 $Q_1 Q_0$（现态，无 `+`），右列标 $Q_1^{+}Q_0^{+}$（次态，有 `+`），列头定死之后再往里填值。
2. **下笔前自问一句**：这一列是「**现在已经存着的**」还是「**时钟沿之后才变成的**」？现在的放左、之后的放右。
3. **检查箭头方向**：状态转移图 / 表里所有箭头都是 present → next，发现 $Q^{+}$ 在左边 = 立刻警觉写反了。

## 🔁 同类题预警

- **状态转移表（state transition table）**、**激励表（excitation table，反求 D/T/JK flip-flop 输入）**、**Moore / Mealy FSM 设计题** 全靠这个方向。
- 一旦现态/次态反了，后面推出来的 flip-flop 输入方程、输出方程会**整片连环错**，且很难自查 —— 所以源头列头一定要先标对。

# 数字电路期中复习 02：如何复习

这份不是知识点笔记，而是 **复习路线图**。  
目标是回答 3 个问题：

1. **先复习什么最值**
2. **每一块具体怎么练**
3. **手头这些作业 / slides / 往年题怎么用**

---

## 1. 先说结论：这门期中最有效的复习顺序

按你现有资料来看，最合理的顺序是：

1. **数制、补码、overflow**
2. **逻辑门、真值表、布尔代数**
3. **SOP / POS / minterm / maxterm / K-map**
4. **NAND-only / NOR-only / 由表达式画电路**
5. **组合逻辑模块：adder / encoder / decoder / comparator / MUX**
6. **延迟、active-low、器件词汇**
7. **MOSFET / CMOS / TTL / open-drain / fan-out**

为什么这样排：

- `HW1` 前半、`22Spring`、`24Spring` 开头都先考 **conversion / complement / arithmetic**
- `HW1 + HW2 + 22Spring + 24Spring` 的主体都在 **Boolean simplification / SOP / POS / K-map**
- `HW3 + lecture_7` 才开始集中考 **combinational building blocks**
- `24Spring` 还碰到了 **MOSFET / NMOS / PMOS / XNOR transistor-level**，所以这块不能完全不看，但优先级低于前面主线

## 3. 每一块应该怎么复习

不要“从头看一遍 slide 然后感觉自己懂了”。  
最有效的节奏是四步循环：

1. **看 10~20 分钟 slide / 术语表**
2. **闭卷写出核心定义或公式**
3. **立刻做 2~4 道对应题**
4. **把错题归因成固定错误类型**

---

## 4. 各模块的具体复习方法

### 4.1 数制与补码：要练“速度”和“位宽意识”

复习动作：

1. 连续做进制转换
2. 连续做 `+ / -` 的补码计算
3. 每做一题都强迫自己先写：
   - 位宽
   - 表示范围
   - 是否 overflow

你要养成的固定检查顺序：

1. 题目给没给位宽
2. 是 `sign-magnitude`、`1's complement` 还是 `2's complement`
3. 结果是否超范围
4. 补码加法时最高位进位是否要丢弃

最常见错误：

- 忘了位宽固定
- 把 `1's complement` 和 `2's complement` 混了
- 结果算对了但没写 `overflow`

最适合练的题：

- `HW1` Problem 1, 2
- `22Spring` 前 1~4 题
- `24Spring` 前 1~2 题

---

### 4.2 真值表与门图：要练“来回翻译”

复习动作：

1. 看门图写表达式
2. 看表达式写真值表
3. 看真值表反推 SOP

你必须做到：

- 看到 `NAND/NOR/XOR/XNOR` 不犹豫
- 多输入门不出错
- `active-low` 时知道有效不一定是 1

最常见错误：

- 漏掉取反泡泡
- 把 XOR 当 OR
- 真值表行顺序乱掉

最适合练的题：

- `HW1` Problem 4, 6
- `22Spring` 逻辑门真值表题
- `24Spring` 3-input NAND gate

---

### 4.3 SOP / POS / K-map：要练“标准套路”

复习动作：

1. 先手算 `standard SOP`
2. 再手算 `standard POS`
3. 再把它们映射到 K-map
4. 再做 `group 1s`、`group 0s`

你要会的固定套路：

#### 从表达式到 standard SOP

- 缺哪个变量，就乘 `(X + X')`
- 展开后每一项都含所有变量

#### 从表达式到 standard POS

- 缺哪个变量，就加上乘积等价展开
- 或者先从真值表 / SOP 反推

#### K-map 化简

- 最简 SOP：**圈 1**
- 最简 POS：**圈 0**
- 分组大小优先取 `1, 2, 4, 8, ...`
- 可以跨边界相邻
- `don't-care` 只在有利于化简时使用

最常见错误：

- Gray code 顺序填错 K-map
- 把 `standard SOP` 和 `minimal SOP` 混掉
- 圈分组时贪小不贪大
- POS 题还在圈 1

最适合练的题：

- `HW2` Problem 1~4
- `22Spring` 第 11, 14 题
- `24Spring` 第 7, 8 题

---

### 4.4 NAND-only / NOR-only：要练“看结构变形”

复习动作：

1. 先把式子整理成 SOP 或 POS
2. 再用 `DeMorgan` 做双重取反
3. 再用 bubble 图理解每一层门

你要会的核心思想：

- `NAND-only` 最顺手的是从 **SOP** 出发
- `NOR-only` 最顺手的是从 **POS** 出发

最常见错误：

- 直接硬画，没先整理表达式
- 取反层数不清
- 忘了已给 `complemented inputs are available`

最适合练的题：

- `HW1` Problem 7
- `HW2` Problem 6
- `22Spring` Traffic light NAND-only

---

### 4.5 Adders / Encoder / Decoder / Comparator / MUX：要练“功能识别”

这一块不要死背长表达式，先背 **功能 + 接口 + 典型结构**。

#### 加法器

你要会：

- 半加器 / 全加器的输入输出含义
- `Sum` 和 `Cout`
- 为什么 `ripple carry` 慢
- `look-ahead carry` 为什么快

#### Priority Encoder

你要会：

- “谁优先级最高”
- `valid bit` 表示什么
- 多个输入同时为 1 时如何处理

#### Decoder

你要会：

- `n-to-2^n`
- `active-high` vs `active-low`
- `enable input`
- minterm 和 decoder output 的对应关系

#### Comparator

你要会：

- `A=B`
- `A>B`
- `A<B`
- 比较高位优先

#### MUX

你要会：

- `select lines`
- 根据选择端决定输出哪一路
- 如何用 MUX 实现布尔函数

最常见错误：

- 不知道 `active-low output` 意味着选中时输出 0
- priority encoder 忽略“高优先级屏蔽低优先级”
- comparator 没有从高位开始判断
- MUX 题不会把变量分成 `select` 和 `data inputs`

最适合练的题：

- `HW3` Problem 1~6
- `lecture_7_part1~3`
- `22Spring` full adder / ripple carry / look-ahead carry

## 7. 考前最后 2 小时怎么用

不要再看新 slide。只做：

1. 快速过一遍术语表
2. 手写一遍：
   - `two's complement`
   - `overflow`
   - `standard SOP / POS`
   - `K-map`
   - `half-adder / full-adder`
   - `MUX / priority encoder / decoder / comparator`
3. 看自己的错题，不看新题

你在这两小时里最该确认的是：

- 自己会不会因为英文看不懂而卡住
- 自己会不会因为步骤不规范而丢分

---

## 8. 复习时的判定标准

不要问“我是不是看懂了”，要问：

1. 我能不能闭卷写出定义？
2. 我能不能 3 分钟内判断题目属于哪一类？
3. 我能不能不看答案做出一题同类题？
4. 我能不能用英文术语说出这题在考什么？

只要这 4 个里有一个不行，就说明这块还没复习完。

---

## 9. 建议你接下来的实际顺序

下一步建议这样推进：

1. 先复习 `数制 + 补码 + overflow`
2. 再复习 `SOP / POS / minterm / maxterm / K-map`
3. 再复习 `NAND-only / NOR-only`
4. 最后整理 `adder / encoder / decoder / comparator / MUX`

这是最稳的路径，因为它和：

- 作业顺序一致
- slides 顺序一致
- 往年题主要得分点也一致

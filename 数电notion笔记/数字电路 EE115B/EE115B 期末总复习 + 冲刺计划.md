### A. Before midterm（约 30%）

| 模块 | 必会 | 常见坑 |

| Number system conversions | binary / octal / hex 互转；signed number；补码；小数转换 | 位权看错、补码符号位处理错 |

### B. Core 70%：Combinational logic

| 模块 | 必会 | 老师最可能怎么考 |
| --- | --- | --- |
| Adder / Comparator | half/full adder、cascade、比较器输出逻辑 | 给模块图让你写 truth table / expression，或反过来设计 |
| Encoder / Decoder | 功能、优先级、active-high / active-low、应用 | 看图判断输入输出含义 |
| MUX / DEMUX | 选择线和数据线关系；用 MUX 实现逻辑函数 | 实现题、模块替代题 |
| Combinational analysis | 电路 → 表达式 → 真值表 → 最简式 | 综合大题 |
| Hazard | static hazard 概念、为什么产生、怎么消掉 | 看 K-map / timing 找 hazard |

### C. Core 70%：Sequential logic

| 模块 | 必会 | 常见坑 |
| --- | --- | --- |
| FSM | Moore / Mealy；state diagram / table；state encoding；next-state / output logic | present state / next state / output 写乱 |
| Latches / Flip-flops | SR / gated D latch；D/T/JK FF；level-sensitive vs edge-triggered | latch 和 FF 混淆 |
| Shift register / Counter | 状态演化、shift direction、mod-N counter | 方向写反、漏状态、边界错 |
| Memory | RAM / ROM / PLD；memory expansion；address lines / data width / capacity | 先算地址、后算位宽这一步经常乱 |

### D. Core 70%：ADC / DAC

| 模块 | 必会 | 关键词 |
| --- | --- | --- |
| DAC | weighted-resistor、R-2R ladder、概念与优缺点 | resolution、linearity、实现复杂度 |
| ADC | flash、counter / digital ramp、SAR 的基本思想与 trade-off | speed vs hardware complexity vs accuracy |
| Quantization | LSB、量化误差、分辨率 | step size、误差上界 |
| Preconditioning circuits | 为什么需要放大 / 采样保持 / 预处理 | 概念题 |

### E. HDL / FPGA / grammar（只考概念，不考你手写程序）

- **不会直接要求写 Verilog / VHDL program**。
- 但要会辨认：`module / entity`、`always / process`、组合逻辑描述 vs 时序逻辑描述、simulation vs synthesis、FPGA 基本概念。
- 换句话说：**考“你懂不懂语言和硬件的关系”，不是考你现场敲代码。**

## 5️⃣ 高频易错点 / ADHD 小错误检查清单

1. **carry ≠ overflow**：尤其 binary arithmetic 和 adder 题。
2. **真值表行序从 000 起、从小到大**：写 / 填真值表一律按二进制递增逐行排（000→001→010→011→…），最左变量当 MSB；乱序 / 跳行最容易漏行、对错位，minterm 编号也跟着标错。
3. **K-map 先看 adjacency，再谈最简**：别只顾着找大块，漏掉环绕相邻。
4. **MUX / DEMUX 先确认 selection lines**：很多错都是线序读反。
5. **FSM 先写 present state，再写 input，再写 next state / output**：不要脑内跳步。
6. **Latch vs FF**：一个是 **level-sensitive**，一个是 **edge-triggered**。
7. **Counter 先列状态序列**：别一边算一边猜。
8. **Memory 题先问自己三句**：
    - 有多少 address？
    - 每个 word 几 bit？
    - 一共需要几片芯片？
9. **ADC / DAC 先抓 trade-off**：速度、复杂度、分辨率三件事别混。
10. **Verilog / VHDL 只要抓概念，不要自己吓自己去背整套语法。**

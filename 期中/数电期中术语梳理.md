## 1. 题干常见动作词 Action Verbs

这些词比名词还重要，因为它们决定你要做什么。

| English | 中文 | 看到它时你要做什么 |
| --- | --- | --- |
| convert | 转换 | 进制转换、SOP/POS 转换、补码转换 |
| perform the following calculations | 完成下列计算 | 按题目要求逐步算，通常要写过程 |
| compute | 计算 | 求结果，常见于位运算、布尔值 |
| determine | 确定 | 求表达式、求输出、求真值 |
| derive | 推导 | 从电路图/定义一步步写出表达式 |
| simplify | 化简 | 用布尔代数、K-map、DeMorgan 化简 |
| minimize | 最小化 | 化到最简形式，通常是 minimal SOP/POS |
| implement | 实现 | 用指定器件或门电路把函数画出来 |
| construct | 构造 | 用多个芯片/模块拼出更大的模块 |
| complete the truth table | 补全真值表 | 把所有输入组合对应输出写完整 |
| list the minterms | 列出最小项 | 写成 `m(...)` 的形式 |
| state whether overflow occurs | 说明是否溢出 | 算完还要判断 `overflow` |
| show the groupings | 标出分组 | K-map 里把圈出来的 grouping 画清楚 |
| show necessary steps | 展示必要步骤 | 不能只写答案 |
| prove | 证明 | 用布尔代数或逻辑推演，不是只凭感觉 |
| assume | 假设 | 注意题目额外规定，如 `Y2 is the MSB` |
| treat ... as don't-care | 视为无关项 | K-map 中可以按 `X` 用来帮助化简 |
| using only NAND gates | 只能用 NAND 门 | 题目在考 universal gate conversion |
| draw the schematic | 画原理图 | 画门级连接，不只是写公式 |

## 2. 基础信号与数制 Basic Signal / Number Terms

### 2.1 信号与电平

| English | 中文 | 备注 |
| --- | --- | --- |
| analog quantity | 模拟量 | 连续变化 |
| digital quantity | 数字量 | 离散的 0/1 |
| logic level | 逻辑电平 | `logic 0` / `logic 1` |
| threshold voltage | 阈值电压 | 判断 0/1 的门限 |
| rise time | 上升时间 | 波形从低到高的时间 |
| fall time | 下降时间 | 波形从高到低的时间 |
| leading edge | 前沿 / 上升沿 | 通常就是信号开始变化的边沿 |
| trailing edge | 后沿 / 下降沿 | 与 leading edge 对应 |
| pulse width | 脉冲宽度 | 一个脉冲持续多久 |
| frequency | 频率 | 单位时间内重复次数 |
| serial | 串行 | 一位一位传 |
| parallel | 并行 | 多位同时传 |
| CMOS | 互补金属氧化物半导体逻辑 | 课程里经常和 TTL 对比 |
| TTL | 晶体管-晶体管逻辑 | 也是常见逻辑家族 |

### 2.2 数制与编码

| --- | --- | --- |

| octal | 八进制 | base-8 |
| decimal | 十进制 | base-10 |
| hexadecimal | 十六进制 | base-16 |

| most significant bit (MSB) | 最高有效位 | 最左边，高位 |

| signed number | 有符号数 | 有正负号 |

| sign-magnitude form | 原码 / 符号-数值表示 | `sign + magnitude` |
| ones' complement | 反码 | 全部取反 |
| two's complement | 补码 | 反码再加一 |
| bitwise NOT | 按位取反 | 每一位单独取反 |
| overflow | 溢出 | 超出当前位宽可表示范围 |
| BCD (Binary Coded Decimal) | 二进制编码十进制 | 默认常指 `8421 BCD` |

## 3. 逻辑门与布尔代数 Logic Gates / Boolean Algebra

### 3.1 逻辑门

| English | 中文 | 备注 |
| --- | --- | --- |
| logic gate | 逻辑门 | 数字电路基本单元 |
| inverter | 反相器 | 就是 `NOT gate` |
| AND gate | 与门 | 输出是乘积关系 |
| OR gate | 或门 | 输出是求和关系 |
| NOT gate | 非门 | 取反 |
| NAND gate | 与非门 | `NOT(AND)` |
| NOR gate | 或非门 | `NOT(OR)` |
| XOR gate (exclusive-OR) | 异或门 | 不同为 1 |
| XNOR gate (exclusive-NOR) | 同或门 / 异或非门 | 相同为 1 |
| multi-input gate | 多输入门 | 如 `4-input NAND` |
| universal gate | 通用门 | NAND / NOR 可实现任意逻辑 |

### 3.2 布尔表达与真值表

| English | 中文 | 备注 |
| --- | --- | --- |
| Boolean expression | 布尔表达式 | 最常见的写法 |
| Boolean statement | 布尔语句 | slide 里常见同义表达 |
| truth table | 真值表 | 输入到输出的完整枚举 |
| literal | 文字 / 变量或其反 | 如 `A`、`A'` |
| product term | 乘积项 | 与项，例如 `ABC'` |
| sum term | 和项 | 或项，例如 `A+B+C'` |
| complemented variable | 取反变量 | 如 `A'` |
| uncomplemented variable | 未取反变量 | 如 `A` |

### 3.3 高频化简术语

| English | 中文 | 备注 |
| --- | --- | --- |
| Sum of Products (SOP) | 与或式 | 若干 product term 相加 |
| standard SOP | 标准与或式 | 每个 minterm 都包含所有变量 |
| Product of Sums (POS) | 或与式 | 若干 sum term 相乘 |
| standard POS | 标准或与式 | 每个 maxterm 都包含所有变量 |
| minterm | 最小项 | 在某一组输入下唯一为 1 |
| maxterm | 最大项 | 在某一组输入下唯一为 0 |
| Karnaugh map (K-map) | 卡诺图 | 期中高频 |
| grouping | 分组 | K-map 里圈 1 或圈 0 |
| don't-care condition | 无关项 | 可以当 0 也可以当 1 |
| DeMorgan's theorem | 德摩根定律 | NAND/NOR 化简必备 |
| consensus theorem | 一致性定理 | 如 `AB + A'C + BC = AB + A'C` |
| Quine-McCluskey method | Quine-McCluskey 化简法 | 比 K-map 更系统 |
| prime implicant (PI) | 主蕴含项 | Q-M 法常见 |
| essential prime implicant | 本质主蕴含项 | 覆盖不可替代的 minterm |
| bubble-bubble cancellation | 泡泡抵消 | 画门图时很常用 |
| minimal SOP form | 最简 SOP 形式 | 作业和考试高频表述 |
| minimal POS form | 最简 POS 形式 | 与上对应 |

### 3.4 逻辑实现相关

| English | 中文 | 备注 |
| --- | --- | --- |
| combinational logic | 组合逻辑 | 输出只由当前输入决定 |
| AND-OR logic | 与或逻辑 | 对应 SOP 实现 |
| AND-OR-Invert (AOI) logic | 与或非逻辑 | 常对应 POS 实现 |
| NAND-only implementation | 纯 NAND 实现 | 只能用 NAND 门 |
| NOR-only implementation | 纯 NOR 实现 | 只能用 NOR 门 |
| mapped logic circuit | 映射后的逻辑电路 | 题里常让你画 |

## 4. 组合逻辑模块 Combinational Building Blocks

这是近年作业和 slides 最集中的一块，也是你最该背英文名词的一块。

### 4.1 Adders 加法器

| English | 中文 | 备注 |
| --- | --- | --- |
| adder | 加法器 | 总称 |
| half-adder | 半加器 | 只加两个输入位，不处理输入进位 |
| full-adder | 全加器 | 处理 `A, B, Cin` |
| sum | 和位 | 加法结果位 |
| carry | 进位 | 向更高位传递 |
| carry-in (Cin) | 输入进位 | 来自低位 |
| carry-out (Cout) | 输出进位 | 送往高位 |
| parallel adder | 并行加法器 | 多位同时计算 |
| ripple carry adder | 串行进位加法器 / 波纹进位加法器 | 进位一级一级传 |
| worst-case carry propagation delay | 最坏情况下的进位传播延迟 | 题里常问 delay |
| look-ahead carry adder | 超前进位加法器 | 用 `Cg/Cp` 降低延迟 |
| generated carry (Cg) | 产生进位 | 常写成 `Cg = AB` |
| propagated carry (Cp) | 传播进位 | 常写成 `Cp = A + B` |
| cascaded | 级联 | 多个加法器拼更高位 |
| arithmetic logic unit (ALU) | 算术逻辑单元 | adder 是其中一部分 |

### 4.2 Encoder / Decoder / Comparator / MUX

| English | 中文 | 备注 |
| --- | --- | --- |
| encoder | 编码器 | 把一条有效输入编码成二进制输出 |
| binary encoder | 二进制编码器 | `2^n` 输入到 `n` 输出 |
| decimal-to-BCD encoder | 十进制到 BCD 编码器 | slides 明确讲过 |
| priority encoder | 优先编码器 | 多个输入同时为 1 时按优先级决定 |
| valid bit | 有效位 | 表示“是否至少有一个输入有效” |
| decoder | 译码器 / 解码器 | 把输入码字展开成某条输出 |
| active-high output | 高电平有效输出 | 选中时输出 1 |
| active-low output | 低电平有效输出 | 选中时输出 0 |
| enable input | 使能输入 | 没有 enable 通常不工作 |
| strobe | 选通 / 门控信号 | 某些芯片手册里常用 |
| asserted | 被置为有效 | 不一定等于 1，要看 active-high 还是 active-low |
| comparator | 比较器 | 比较两个数大小 |
| equality output | 相等输出 | `A = B` |
| greater-than output | 大于输出 | `A > B` |
| less-than output | 小于输出 | `A < B` |
| multiplexer (MUX) | 多路复用器 / 多路选择器 | 从多个输入中选一路输出 |
| data selector | 数据选择器 | MUX 的同义表达 |
| select line | 选择线 | 决定选哪一路 |
| demultiplexer (DEMUX) | 解复用器 | 一路输入分发到多路输出 |
| data distributor | 数据分配器 | DEMUX 的同义表达 |

### 4.3 其他常考模块词

| English | 中文 | 备注 |
| --- | --- | --- |
| majority voting circuit | 多数表决电路 | `at least two inputs are 1` |
| MUX-based implementation | 基于 MUX 的实现 | HW3 直接考了 |

## 7. 容易混淆的一组词

| A | B | 区别 |
| --- | --- | --- |
| encoder | decoder | `encoder` 是“压缩/编码”，`decoder` 是“展开/译码” |
| XOR | XNOR | `XOR` 不同为 1，`XNOR` 相同为 1 |
| half-adder | full-adder | `half` 没有 `Cin`，`full` 有 `Cin` |
| ripple carry | look-ahead carry | 前者慢在逐级传进位，后者快在提前算进位 |
| active-high | active-low | 有效电平一个是 1，一个是 0 |
| standard SOP | minimal SOP | 前者规范展开，后者追求最简 |
| sign-magnitude | ones' complement / two's complement | 都是有符号表示，但考试最常用的是 `two's complement` |
| MUX | DEMUX | `MUX` 多选一输出，`DEMUX` 一路分发到多路 |
| asserted | logic 1 | 不一定一样；若是 `active-low`，asserted 可能是 0 |

## 8. 考试里常见整句，直接记意思

### 8.1 化简类

- `simplify it to its minimal SOP form`
  化简成最简与或式。

- `convert the Boolean function to standard SOP form`
  把布尔函数展开成标准与或式。

- `list the minterms`
  列出所有最小项，通常写成 `m(...)`。

- `apply DeMorgan's theorems`
  应用德摩根定律。

- `show your groupings in the Karnaugh map`
  在卡诺图上把分组圈出来。

### 8.2 电路实现类

- `implement the function using one 4:1 MUX`
  只能用一个 4 选 1 多路复用器实现。

- `using only NAND gates`
  只能用 NAND 门，通常要做门级等效变换。

- `draw the final schematic`
  画最终电路图。

- `construct a 4-to-16 decoder using two 3-to-8 decoders`
  用两个 3-8 译码器拼一个 4-16 译码器。

### 8.3 编码器/比较器类

- `highest-indexed input has the highest priority`
  编号最大的输入优先级最高。

- `with an input valid bit`
  还要额外设计一个“输入有效位”。

- `derive the boolean expressions for equal, greater, and less outputs`
  推导相等、大于、小于三个输出的布尔表达式。

### 8.4 算术类

- `perform the following calculations using 8-bit two's-complement representation`
  用 8 位补码完成以下运算。

- `state whether overflow occurs`
  最后别忘了判断是否溢出。

- `provide two input numbers that gives the worst delay`
  找出让延迟最糟糕的输入情况。

## 10. 下一步复习建议

1. `数制与补码`
